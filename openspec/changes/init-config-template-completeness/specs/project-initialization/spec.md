## MODIFIED Requirements

### Requirement: Init generates default config.toml

The system SHALL create `.git-paw/config.toml` with sensible defaults and commented-out example fields when no config exists. The generated config SHALL include an active `worktree_placement = "child"` field so that new repositories use the contained (in-repo) worktree layout by default. The generated template SHALL present a commented example for every config section documented in the configuration reference — so the template is a complete, discoverable schema rather than a subset — including `[mcp]`, `[layout]`, `[broker.watcher]`, `[supervisor.auto_approve]`, `[supervisor.learnings_config]`, and `[governance]` in addition to the base fields and the `[specs]`, `[logging]`, `[dashboard]`, `[broker]`, `[supervisor]`, and `[opsx]` sections.

#### Scenario: Config created with defaults
- **WHEN** `git paw init` is run and no `.git-paw/config.toml` exists
- **THEN** `.git-paw/config.toml` SHALL be created with `default_cli` and `mouse` fields and commented examples for `default_spec_cli`, `branch_prefix`, `[specs]`, and `[logging]`

#### Scenario: Template documents every supported config section
- **WHEN** `git paw init` generates the default `.git-paw/config.toml`
- **THEN** the template SHALL include a commented example stanza for each of `[mcp]`, `[layout]`, `[broker.watcher]`, `[supervisor.auto_approve]`, `[supervisor.learnings_config]`, and `[governance]`
- **AND** every config section documented in `docs/src/configuration` SHALL have a corresponding commented example in the template

#### Scenario: Added stanzas are commented and do not change defaults
- **WHEN** a config generated by `git paw init` is loaded without any user edits
- **THEN** the added stanzas SHALL all be commented, so the effective configuration is identical to the previous template's defaults

#### Scenario: Config created with child worktree placement
- **WHEN** `git paw init` is run and no `.git-paw/config.toml` exists
- **THEN** `.git-paw/config.toml` SHALL contain `worktree_placement = "child"`

#### Scenario: Existing config is not overwritten
- **WHEN** `git paw init` is run and `.git-paw/config.toml` already exists
- **THEN** the existing config SHALL NOT be modified
